# Open Source Funding Platforms

## Overview

Platforms for receiving direct financial support for open source projects.

## Comparison Matrix

| Platform | Model | Potential Revenue | Setup Difficulty | Fees |
|----------|-------|-------------------|------------------|------|
| GitHub Sponsors | Direct sponsorship | $100-$10K/mo | Easy | 0% |
| Open Collective | Fiscal host | $1K-$50K/mo | Medium | 5-10% |
| Patreon | Membership | $100-$5K/mo | Easy | 5-12% |
| Ko-fi | One-time + monthly | $50-$2K/mo | Easy | 0-5% |
| Polar.sh | Bounties + subscriptions | Variable | Easy | 5% |
| Thanks.dev | Dependency-based | $10-$1K/mo | Easy | 10% |
| Liberapay | Recurring donations | $50-$1K/mo | Medium | 0% |

## Platform Details

### 1. GitHub Sponsors

**Best For**: GitHub-hosted projects with active development

**Features**:
- Zero fees
- Direct integration with GitHub
- Tier-based rewards
- One-time and recurring
- Corporate matching

**Setup**:
1. Enable GitHub Sponsors in settings
2. Create sponsor tiers
3. Add goals and rewards
4. Promote in README

**Tiers Suggestion**:
- $5/mo: Supporter badge
- $25/mo: Name in CONTRIBUTORS
- $100/mo: Priority support
- $500/mo: Monthly consulting call
- Custom: Enterprise sponsorship

### 2. Open Collective

**Best For**: Projects needing fiscal hosting and transparent finances

**Features**:
- Full financial transparency
- Fiscal hosting
- Expense management
- Multiple contributors
- Grant applications

**Setup**:
1. Create collective
2. Choose fiscal host
3. Set up payment methods
4. Define expense policy

**Use Cases**:
- Projects without legal entity
- Community-governed projects
- Need for expense reimbursement

### 3. Polar.sh

**Best For**: Modern approach with bounties and subscriptions

**Features**:
- Issue bounties
- Subscription tiers
- Newsletter integration
- Discord benefits
- Modern UI

**Setup**:
1. Connect GitHub
2. Create subscription tiers
3. Enable issue funding
4. Promote to community

### 4. Thanks.dev

**Best For**: Projects with many dependents

**Features**:
- Automated sponsorship requests
- Based on project usage
- Low effort
- Passive income

**Setup**:
1. Register project
2. Add badge to README
3. Automatic dependency analysis

## Best Practices

### 1. Transparency

- Show what funding supports
- Share progress updates
- List sponsors publicly

### 2. Value Proposition

- Clear benefits per tier
- Regular updates for sponsors
- Exclusive content/access

### 3. Marketing

- Add badges to README
- Mention in release notes
- Share impact stories
- Tweet about sponsors

### 4. Sustainability

- Multiple funding sources
- Don't rely on single sponsor
- Build recurring revenue
- Save for stability

## Revenue Potential

- GitHub Sponsors: $200/mo
- Open Collective: $100/mo
- **Total**: ~$3,600/year

### Moderate (Year 2)

- GitHub Sponsors: $1,000/mo
- Open Collective: $500/mo
- Polar.sh bounties: $200/mo
- **Total**: ~$20,400/year

### Ambitious (Year 3+)

- GitHub Sponsors: $5,000/mo
- Open Collective: $2,000/mo
- Corporate sponsors: $3,000/mo
- **Total**: ~$120,000/year

## Recommendations for Open-Jarvis

1. **Immediate**: Setup GitHub Sponsors
   - Free, integrated with GitHub
   - Add to README immediately

2. **Short-term**: Join Open Collective
   - Apply to Open Source Collective as fiscal host
   - Good for transparency

3. **Medium-term**: Add Polar.sh
   - For issue bounties
   - Engages contributors

4. **Long-term**: Build towards SaaS
   - Hosted version of Open-Jarvis
   - Enterprise features

## Tax Considerations

- **US**: Report as income (1099 or W-2 from platform)
- **EU**: VAT may apply
- **Consulting**: May need business license

**Recommendation**: Consult tax professional when revenue > $1,000/month
