# PRD Generation Workflow

This workflow guides the creation of a comprehensive Product Requirements Document (PRD).

## 1. Context Gathering
1. **Stakeholder Input**:
   - Collect raw requirements, notes, and goals
   - Identify target audience and key constraints

2. **Market Research**:
   - Analyze competitors and market gaps
   - Define the "North Star" metric

## 2. Drafting the PRD
1. **Structure Setup**:
   - Initialize the 16-section PRD template
   - Fill in Executive Summary and Goals

2. **Functional Requirements**:
   - Draft User Stories (As a... I want to... So that...)
   - Define Acceptance Criteria (Gherkin/Bulleted)
   - Prioritize (MoSCoW method)

3. **Technical Requirements**:
   - Define Architecture and Tech Stack
   - Specify API endpoints and Data Models
   - Define Non-functional requirements (Performance, Security)

4. **UX/UI Requirements**:
   - Describe User Flows
   - Link to Design System/Wireframes

## 3. Validation & Refinement
1. **Iterative Review**:
   - Apply the "Iterative Validation Loop"
   - Check for ambiguity and edge cases

2. **Risk Assessment**:
   - Identify potential risks and mitigation strategies

3. **Success Metrics**:
   - Define KPIs and launch criteria

## 4. Finalization
1. **Formatting**:
   - Ensure clear Markdown formatting
   - Add Glossary for technical terms
