# UI/UX Research & Design Workflow

This workflow guides you through a complete UI/UX research and design process, from viewport analysis to design system creation.

## 1. Research Phase
1. **Viewport Analysis**:
   - Identify target devices (Mobile, Tablet, Desktop, Ultrawide)
   - Define breakpoints (e.g., 375px, 768px, 1024px, 1440px)
   - Analyze industry trends for the specific domain

2. **Competitive Analysis**:
   - Identify top 3-5 competitors
   - Analyze their UI patterns, strengths, and weaknesses
   - Identify opportunities for differentiation

3. **User Persona Definition**:
   - Define primary and secondary user personas
   - Map out user goals and pain points

## 2. Design System Foundation
1. **Color System**:
   - Define primary, secondary, and accent colors
   - Create semantic tokens (surface, text, border, status)
   - Define dark mode palette

2. **Typography**:
   - Select typefaces (Header, Body, Mono)
   - Define type scale (Size, Weight, Line Height)

3. **Layout & Spacing**:
   - Define base unit (e.g., 4px or 8px)
   - Define spacing tokens (xs, sm, md, lg, xl)
   - Define grid system (columns, gutters, margins)

## 3. Component Design
1. **Core Components**:
   - Buttons (Primary, Secondary, Ghost, Destructive)
   - Inputs (Text, Select, Checkbox, Radio)
   - Navigation (Navbar, Sidebar, Tabs)

2. **Feedback Components**:
   - Alerts/Toasts
   - Modals/Dialogs
   - Empty States
   - Loading States (Skeleton, Spinner)

## 4. Key Screen Design
1. **Wireframing**:
   - Create ASCII/Low-fi wireframes for critical flows
   - Validate layout against user goals

2. **High-Fidelity Design**:
   - Apply design system to wireframes
   - Ensure accessibility (Contrast, Touch Targets)

## 5. Handoff & Documentation
1. **Design Specs**:
   - Document all tokens and components
   - Provide implementation guidelines for developers

2. **Asset Export**:
   - Prepare icons and images for production
